# Copilot for Product Managers — Lab Files

Hands-on lab exercises for the **GitHub Copilot for Product Managers** training programme. Each lab is designed for pairs, builds on the previous day's output, and integrates GitHub Copilot Chat with Azure DevOps (ADO).

## Labs

| Day | Lab | Focus |
|-----|-----|-------|
| 1 | [Day1_Lab.md](Day1_Lab.md) | Idea → Problem Statement → Personas → JTBD → Metrics → ADO Epic |
| 2 | Day2_Lab.md | Feature decomposition from Epic |
| 3 | Day3_Lab.md | User stories and acceptance criteria |
| 4 | Day4_Lab.md | Sprint planning and backlog refinement |
| 5 | Day5_Lab.md | Retrospective and metrics review |

## Prerequisites

- GitHub Copilot Chat — signed in and active
- Access to the Azure DevOps sandbox/training project
- A real idea from your own product backlog

## Prompting Pattern

All labs use the **CGCO** pattern:

| Component | Description |
|-----------|-------------|
| **C**ontext | Your product, team, users, and current state |
| **G**oal | What you want Copilot to produce |
| **C**onstraints | Timeline, tech, team size, compliance |
| **O**utput format | Exact structure you expect back |

## How to Use These Labs

1. Work in pairs throughout the week.
2. Use your own real backlog items — avoid generic examples.
3. Each day's output feeds directly into the next day's lab.
4. Log all artefacts (Epics, Features, Stories) into the ADO sandbox project as you go.
